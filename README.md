# Compiler-Competition-2025
## IR定义&生成部分
### 核心思路
先参考往年队伍代码，实现与手写前端AST的对接，运行生成IR代码。
成功生成之后修改完善IR。

## 生成IR注意点：
只重载visit是不是会导致由于我们无法提前知晓节点对象所属的类， 所以重载机制无法执行正确的方法。 方法会将 节点基类作为输入参数的默认类型。

## RiscV汇编示例
```riscv
main:
        addi    sp, sp, -32 #sp为栈指针，初始指向高地址，先向下移动32字节，即开辟32字节空间
        sw      ra, 28(sp) #ra为存储函数调用的返回地址的寄存器，ra具体存储sp当前地址向上移动28的地址
        sw      s0, 24(sp) #保存调用者的s0到sp+24
        addi    s0, sp, 32 #这里将s0作为帧指针，指向栈帧顶部
        li      a0, 0 #加载立即数0到a0寄存器，a0寄存器存储函数返回值或函数参数0，对应变量i
        sw      a0, -12(s0) #将a0寄存器的值存储到s0寄存器偏移后指向的地址，即存储0到[s0-12]
        li      a0, 5 #加载立即数5到a0寄存器，对应变量a
        sw      a0, -16(s0) #存储5到[s0-16]
        lw      a0, -16(s0) #加载a的值到a0
        li      a1, 10 #加载立即数10到a1寄存器，临时常数
        blt     a0, a1, .LBB0_3 #如果a0 < a1，跳转到.LBB0_3
        j       .LBB0_1 #跳转到.LBB0_1
.LBB0_1:
        lw      a1, -16(s0) #加载a的值到a1
        li      a0, 19 #加载立即数19到a0
        blt     a0, a1, .LBB0_3 #如果a0 < a1，跳转到.LBB0_3
        j       .LBB0_2 #跳转到.LBB0_2
.LBB0_2:
        lw      a0, -16(s0) 
        addi    a0, a0, 1 #a0++
        sw      a0, -16(s0)
        j       .LBB0_4
.LBB0_3:
        lw      a0, -16(s0)
        addi    a0, a0, -1 #a0--
        sw      a0, -16(s0)
        j       .LBB0_4
.LBB0_4:
        li      a0, 0 #i=0
        sw      a0, -20(s0) #0存入地址s0-20，对应变量i
        lw      a0, -16(s0) #加载变量a值
        sw      a0, -24(s0) #存入地址s0-24，对应变量b
        j       .LBB0_5
.LBB0_5:
        lw      a1, -20(s0) #加载i值
        li      a0, 9 #加载立即数9到a0寄存器
        blt     a0, a1, .LBB0_7 #如果a0 < a1，跳转
        j       .LBB0_6
.LBB0_6:
        lw      a0, -24(s0)
        addi    a0, a0, 1
        sw      a0, -24(s0)
        lw      a0, -20(s0)
        addi    a0, a0, 1
        sw      a0, -20(s0)
        j       .LBB0_5
.LBB0_7:
        li      a0, 0 #加载返回值0
        lw      ra, 28(sp) #恢复返回地址
        lw      s0, 24(sp) #恢复调用者的s0
        addi    sp, sp, 32 #释放栈空间
        ret
```
